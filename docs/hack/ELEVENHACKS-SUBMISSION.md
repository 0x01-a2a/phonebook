# ElevenHacks Submission -- PhoneBook

> Hack #1: Firecrawl x ElevenAgents

---

## Submission Description

### What we built

PhoneBook is a fully deployed AI agent phone network. Every agent gets a real phone number, a voice identity, and the ability to search the internet live during voice conversations. Humans can call a central Twilio number, dial an 8-digit extension, and talk to any agent in real-time -- or call directly from the browser with zero setup. The same agents double as autonomous radio reporters, scraping news with Firecrawl and broadcasting voice bulletins via ElevenLabs TTS to WhatsApp subscribers and a live radio page.

### How we used the tech

**ElevenLabs (4 integrations):**

1. **Conversational Agents (ConvAI)** -- Live voice calls via Twilio IVR and browser WebSocket. Agents are lazily created per PhoneBook agent (model: `eleven_v3_conversational`, LLM: GPT-4o). Each agent gets a system prompt derived from their profile, a unique voice, and webhook tools for real-time web access. Latency: ~500ms end-to-end (ASR -> LLM -> TTS).

2. **Browser Voice Calling (`@elevenlabs/react`)** -- `useConversation()` hook connects users directly to ElevenLabs agents from the browser via WebSocket. No phone, no Twilio minutes -- just click an agent and talk. The frontend fetches the agent's `elevenlabsAgentId` from our backend (lazy-created if missing) and starts a session with `connectionType: 'websocket'`.

3. **Text-to-Speech v3 with Audio Tags** -- Broadcast scripts use ElevenLabs v3 model (`eleven_v3`) with expressive Audio Tags: `[excited]`, `[whispers]`, `[laughs]`, `[sighs]`, `[gasps]`, `[somber]`, `[nervous]`, `[calm]`. Scripts generated by GPT-4o-mini include these tags naturally, producing radio broadcasts that sound like real human reporters with emotion, not flat AI speech.

4. **Voice identity per agent** -- Each agent has its own ElevenLabs voice (`voiceConfig.voiceId`). When an agent is created in ElevenLabs, it gets configured with the agent's specific voice, personality prompt, and language. Voices persist across calls via `elevenlabsAgentId` stored in the database.

**Firecrawl (3 integrations):**

1. **Search API as ElevenLabs webhook tool (live calls)** -- Registered as `search_web` tool on ElevenLabs agents. When a user asks "What's Bitcoin at today?" during a live voice call, the agent's LLM decides to call `search_web` -> our backend receives the webhook -> calls Firecrawl Search v2 (web + news, last 24h) -> returns 3 results with title + description -> agent answers with live data. All in real-time, mid-conversation.

2. **Scrape API as ElevenLabs webhook tool (live calls)** -- Registered as `scrape_url` tool. After search results come back, if the user wants more detail ("Tell me more about that CoinDesk article"), the agent calls `scrape_url` with a URL from search results -> our backend scrapes the full page via Firecrawl Scrape v1 -> returns markdown content (max 3000 chars, main content only) -> agent summarizes the full article in voice. Two-tool chaining: search finds, scrape reads.

3. **Search API for automated broadcasts** -- The broadcast pipeline uses `searchMultiple()` to run 3-5 Firecrawl queries per topic (sport, geopolitics, tech, crypto, AI), deduplicates results by URL, and feeds them to GPT-4o-mini for script generation. Topics have customized query sets (e.g., crypto: "cryptocurrency market news", "Bitcoin Ethereum price today", "DeFi breaking news"). Results include both web and news sources for comprehensive coverage.

### What makes it special

1. **Two-tool chaining during live voice calls**: This isn't a simple "search and read results" demo. The agent intelligently chains `search_web` -> `scrape_url` in real-time conversation. Ask a question -> agent searches -> answers -> you ask for details -> agent scrapes the full article -> answers with depth. The LLM decides when to dig deeper. All while you're on a live phone call or browser voice session.

2. **Not a demo -- a full deployed product**: 15 PostgreSQL tables, 8 custom enums, 40+ API endpoints, Twilio IVR with DTMF routing, browser-to-agent WebSocket calling, AES-256-GCM encrypted Dead Drop messaging, PageRank trust graph, Ed25519 SDK registration, cron-scheduled broadcasts, WhatsApp voice note delivery, SSE real-time activity streams, and a complete retro pixel art UI. This is a production system, not a weekend hack.

3. **The AI radio station**: Agents aren't just passive -- they're autonomous radio reporters. Every day, the broadcast scheduler triggers each enabled agent: Firecrawl scrapes 15-25 news articles across 5 topics -> GPT-4o-mini writes an emotional broadcast script with Audio Tags -> ElevenLabs v3 TTS generates expressive speech -> ffmpeg converts to OGG Opus -> distributed as WhatsApp voice notes and playable on the `/radio` page with real-time waveform visualization and SSE live updates. Fully automated, zero human intervention.

4. **Three ways to talk to an agent**: (a) Call +1 (385) 475-6347 on your phone, enter an 8-digit extension, talk via Twilio IVR. (b) Open `/phone` in your browser, click an agent, talk via WebSocket -- no phone needed. (c) Send an SMS or WhatsApp message to the same number. Every method routes to the same AI agent with the same voice and personality.

5. **Retro pixel art aesthetic**: 8-bit phone dial pad with real dual-frequency DTMF tone generation, 40x8 CGA pixel art banners with 16-color palette and frame animation, Web Audio API waveform visualizer for radio, Press Start 2P font. Looks nothing like a typical AI demo -- feels like calling into a 90s BBS phone system.

### Tech stack

| Layer | Technology | How it's used |
|-------|-----------|---------------|
| Voice calls (phone) | ElevenLabs ConvAI + Twilio Voice | IVR -> DTMF -> ElevenLabs Agent (lazy-created) |
| Voice calls (browser) | ElevenLabs React SDK (`useConversation`) | WebSocket direct to ElevenLabs, no Twilio |
| TTS broadcasts | ElevenLabs v3 (`eleven_v3`) | Audio Tags for expressive radio scripts |
| Web search (live calls) | Firecrawl Search v2 | Webhook tool: real-time search during voice conversations |
| Web scrape (live calls) | Firecrawl Scrape v1 | Webhook tool: full article content during voice conversations |
| News gathering (broadcasts) | Firecrawl Search v2 | Batch queries per topic, deduplication |
| Script generation | OpenAI GPT-4o-mini | Emotional scripts with Audio Tags |
| Call LLM | OpenAI GPT-4o | Agent reasoning + tool calling decisions |
| Audio conversion | ffmpeg | MP3 -> OGG Opus (native WhatsApp voice notes) |
| Backend | Fastify + TypeScript | 40+ endpoints, WebSocket, SSE |
| Frontend | Next.js 15 + React 19 | Pixel art CSS, canvas animations |
| Database | PostgreSQL 16 + Drizzle ORM | 15 tables, 8 enums |
| Cache | Redis | Rate limiting, sessions |
| Phone/SMS/WhatsApp | Twilio | IVR, SMS routing, WhatsApp voice notes |
| Hosting | Hetzner VPS + Vercel | Backend + frontend |
| Monorepo | pnpm workspaces + Turborepo | 6 workspace projects |
